# 【図解】会議録画を自動文字起こし＆発言者ごとに分ける方法（無料・完全ローカル）

> **この記事について**
> - 対象：PCに詳しくない方、文字起こしを効率化したい方
> - 所要時間：セットアップ10分、以降は毎回3ステップのみ
> - 費用：完全無料（ソフトウェア・API費用なし）
> - プライバシー：音声データがインターネットに出ない

---

## こんな人に向いています

- 会議・インタビュー・対談の録画を文字起こしするのが大変
- 「誰がいつ何を言ったか」をテキストで残したい
- 音声データを外部サービスにアップしたくない（機密・個人情報が含まれる）
- 月額費用をかけたくない

---

## 完成した文字起こしのイメージ

```
元ファイル : 会議録画_2026年6月.mp4
作成日時   : 2026-06-05 14:32:10
============================================================

【SPEAKER_00】
  00:05.20 - 00:12.80  本日はお集まりいただきありがとうございます。
  00:13.00 - 00:20.10  まず最初の議題ですが、先日の件について...

【SPEAKER_01】
  00:21.50 - 00:35.40  はい、確認しました。問題なく進んでいます。

【SPEAKER_00】
  00:36.10 - 00:48.00  ありがとうございます。では次の議題に移ります。
```

「SPEAKER_00」「SPEAKER_01」は自動で振られます。  
誰がどちらかは出力後に手動でメモ帳で書き換えてください。

---

## 全体の流れ

```
① ダウンロード（1分）
      ↓
② セットアップ.batをダブルクリック（5〜10分・放置OK）
      ↓
③ HuggingFaceに無料登録してトークン取得（5分）
      ↓
④ アプリを起動してMP4を選んでボタンを押す（毎回3ステップ）
      ↓
⑤ テキストファイルが保存される
```

---

## STEP 1：アプリをダウンロードする

GitHubから無料でダウンロードできます。

1. https://github.com/reijiruiji/transcribe-app を開く
2. 緑の「Code」ボタン → 「Download ZIP」をクリック
3. ダウンロードしたZIPファイルを右クリック →「すべて展開」

フォルダを展開すると中にこんなファイルがあります：

```
📁 transcribe-app-main
  ├── ★セットアップ（初回だけここをダブルクリック）.bat  ← これを最初に実行
  ├── ★使い方（まずここを読んで）.txt
  └── _app
      └── transcribe_app.py  ← アプリ本体
```

---

## STEP 2：セットアップ.batを実行する（最初の1回だけ）

`★セットアップ（初回だけここをダブルクリック）.bat` をダブルクリックします。

**黒い画面が開きます。放置してください。**

```
================================================
  Transcribe App  Setup  (5-10 min)
================================================

[1/3] Python 3.12 OK
[2/3] FFmpeg OK
[3/3] Installing whisperx...
...（数分かかります）
Creating desktop shortcut...

================================================
  Setup complete!
  Double-click [transcribe] on your desktop.
================================================
```

「Setup complete!」と出て画面が閉じたら完了です。  
デスクトップに「transcribe」というショートカットが作られています。

> **「セキュリティの警告」が出た場合**：  
> 「詳細情報」→「実行」で進めてください。  
> このファイルはウイルスではありません。Windowsがダウンロードファイルに出す警告です。

> **「Windowsによってパソコンが保護されました」と出た場合**：  
> 同様に「詳細情報」→「実行」をクリックしてください。

---

## STEP 3：HuggingFaceのトークンを取得する（最初の1回だけ）

「話者を識別するAI」を使うための鍵（トークン）が必要です。  
HuggingFaceという無料サービスで発行できます。

### アカウント作成

1. https://huggingface.co を開く
2. 「Sign Up」をクリック
3. メールアドレスとパスワードを設定（クレジットカード不要）
4. 届いたメールのリンクをクリックして認証

### トークンを発行する

1. ログインした状態で右上のアイコンをクリック
2. 「Settings」を選ぶ
3. 左メニューの「Access Tokens」を開く
4. 「New token」をクリック
5. 名前は何でもOK（例：`transcribe`）
6. **権限は以下の1つだけにチェック：**

```
✅ Read access to contents of all public gated repos you can access
```

（他の項目はチェックしない）

7. 「Create token」を押す
8. 表示された `hf_` で始まる文字列をコピーする

> **この権限が必要な理由**：  
> 話者識別に使うAIモデルは「利用規約に同意した人だけ使える」設定になっています。  
> トークンはその同意を確認するためだけに使われます。  
> 音声データはどこにも送られません。

### モデルの利用規約に同意する

以下の2ページを開いて「Agree」を押すだけ（1分で完了）：

- https://huggingface.co/pyannote/speaker-diarization-3.1
- https://huggingface.co/pyannote/segmentation-3.0

---

## STEP 4：アプリを使う（毎回の操作）

デスクトップの「transcribe」をダブルクリックしてアプリを開きます。

### STEP 1（アプリ内）：トークンを入力

先ほどコピーした `hf_` で始まる文字列を貼り付けます。  
2回目以降は自動的に入力されています。

### STEP 2（アプリ内）：ファイルを選ぶ

「ファイルを選ぶ」ボタンを押して、文字起こしたいMP4ファイルを選びます。

対応フォーマット：MP4、MOV、MKV、AVI、MP3、WAV、M4A

### STEP 3（アプリ内）：処理方式を選んで開始

処理方式は「自動」のままでOKです。  
「▶ 文字起こし開始」を押します。

**処理中はアプリのログ欄に進捗が表示されます：**

```
  ファイル : 会議録画.mp4
  処理方式 : CPU（GPUより遅い）
  モデル   : large-v3

【1/4】 音声を読み込んでいます...
【2/4】 文字起こし中...
       完了
【3/4】 タイムスタンプを合わせています...
       完了
【4/4】 発言者を分析中...
       完了

✅ 完了しました！
   保存先: C:\Users\...\文字起こし\会議録画_20260605_143210.txt
```

完了すると「完了！」のポップアップが出ます。  
「保存フォルダを開く」ボタンで結果を確認できます。

## 処理時間の目安

| 動画の長さ | 処理時間（CPU） | 処理時間（GPU） |
|---|---|---|
| 10分 | 約20〜30分 | 約1〜2分 |
| 30分 | 約60〜90分 | 約3〜5分 |
| 60分 | 約2〜3時間 | 約6〜10分 |

CPUの場合、処理中もパソコンは普通に使えます。寝る前に開始して翌朝確認するのがおすすめです。

## よくあるエラーと対処法

### 「セットアップ未完了」のバナーが出る

`★セットアップ.bat` を再実行してください。

### 「トークンエラー」と出る

- `hf_` で始まる文字列が正しく入力されているか確認
- HuggingFaceで新しいトークンを発行して貼り直す

### 「モデルアクセス拒否」と出る

- https://huggingface.co/pyannote/speaker-diarization-3.1
- https://huggingface.co/pyannote/segmentation-3.0

上記2ページで「Agree」を押していない可能性があります。

### 処理が途中で止まる（CPUで長時間待っている場合）

正常です。CPUでの処理は時間がかかります。ログ欄に何か出ていれば処理中です。

### 結果が正確じゃない

- 録音の音量が小さい・雑音が多い場合は精度が落ちます
- マイクに近い環境で録音するほど精度が上がります

## 出力ファイルの編集方法

保存されるテキストファイルをメモ帳で開いて編集できます。

`SPEAKER_00` → `田中さん` のように書き換えると議事録として使いやすくなります。
